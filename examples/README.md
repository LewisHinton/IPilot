# IPilot Examples

These examples show how IPilot behaves across common IP asset workflows. They are written as workflow demos, not as final brand assets.

| Example | Best For | What It Demonstrates |
|---|---|---|
| `tensor-cat-demo.md` | developer / education community assets | sticker concepts, mild technical humor, safety review |
| `campus-coffee-fox-demo.md` | campus or small-brand mascot campaigns | mascot consistency plus platform-aware social concepts |
| `occasion-birthday-demo.md` | birthday and milestone assets | material-type confirmation before occasion-based production |
| `deep-ip-consistency-demo.md` | strict visual consistency work | identity anchors, line language, color ratio, construction rules |
| `reference-image-workflow-demo.md` | existing mascot art with multiple references | source-of-truth selection and conflict resolution |

## How to Use These Examples

1. Pick the example closest to your use case.
2. Compare it with `templates/ip-brief-template.md`.
3. Replace the sample IP with your own owned mascot or brand figure.
4. Ask IPilot to run the same workflow.

Do not copy example mascots as real brand IP. They are placeholders for workflow demonstration.
