# Material Taxonomy

IPilot should not treat all IP assets as the same kind of output. Before generating detailed concepts, classify the requested work into a material type. If the type is unclear, present a short menu and ask the user to confirm.

## Core taxonomy

### 1. IP identity assets

Use when the goal is recognition, introduction, or consistency.

Examples:

- IP introduction card
- mascot profile card
- character reference sheet
- expression sheet
- pose sheet
- signature pose set
- avatar / profile image
- profile banner / header
- brand mascot lockup concept

Ask if missing:

- where this identity asset will be used;
- whether it should be public-facing or internal;
- which features must be treated as canonical.

### 2. Community interaction assets

Use when the goal is chat, community participation, fan engagement, or daily interaction.

Examples:

- sticker pack
- emoji pack
- reaction image
- comment reply card
- welcome card
- fan badge
- contributor badge
- community challenge card
- punchline / catchphrase sticker

Ask if missing:

- target community or platform;
- allowed humor level;
- whether the asset should be cute, sarcastic, comforting, energetic, or ceremonial.

### 3. Meme and humor assets

Use when the user wants shareable, humorous, relatable, or viral IP content.

Examples:

- meme concept pack
- reaction meme template
- four-panel comic
- relatable pain-point meme
- seasonal meme pack
- platform-native joke card
- trend-adapted IP meme

Ask if missing:

- humor boundary;
- forbidden joke types;
- platform and audience;
- whether the meme should be brand-safe, edgy, self-deprecating, or purely cute.

### 4. Digital promotional assets

Use when the goal is announcement, conversion, campaign visibility, launch communication, or event promotion.

Examples:

- campaign poster
- event announcement poster
- product launch card
- registration poster
- social media cover
- carousel card
- short-video cover
- livestream cover
- article header image
- CTA card
- countdown card
- release note visual

Mandatory confirmation:

- target publishing platform;
- platform placement;
- key message;
- CTA;
- date, deadline, price, QR code, or event information if needed.

### 5. Occasion-based IP assets

Use when the asset is tied to a specific date, festival, anniversary, birthday, seasonal moment, or brand milestone.

Examples:

- IP birthday key visual
- IP birthday countdown card
- birthday sticker pack
- anniversary poster
- holiday greeting card
- festival-themed poster
- seasonal campaign visual
- year-end thank-you card
- New Year / Lunar New Year card
- school opening / exam season card
- milestone celebration card

Ask if missing:

- occasion type;
- date or timing;
- platform;
- cultural context;
- tone: celebratory, warm, humorous, premium, ceremonial, cute, or playful;
- whether the output is a single key visual or a series.

### 6. Educational and explanatory assets

Use when the IP represents a knowledge brand, course, technology community, science communicator, or learning product.

Examples:

- concept explainer card
- glossary card
- tutorial thumbnail
- learning progress sticker
- error-state reaction image
- step-by-step visual card
- workshop summary card

Ask if missing:

- learning goal;
- audience expertise level;
- whether the IP should teach, guide, react, or simplify.

### 7. Lifecycle and retention assets

Use when the user needs recurring IP content rather than one-time output.

Examples:

- weekly mood sticker
- monthly calendar card
- user achievement badge
- check-in card
- membership milestone card
- release milestone card
- community ritual card

Ask if missing:

- recurrence cycle;
- user behavior to reinforce;
- whether the asset is for acquisition, retention, or reactivation.

### 8. Physical and tangible materials

Use when the output will be printed, manufactured, distributed, displayed, or sold.

Examples:

- sticker sheet
- badge / pin
- acrylic standee
- keychain
- tote bag graphic
- T-shirt graphic
- postcard
- thank-you card
- package insert
- flyer
- booth poster
- coupon card
- standee
- table tent
- hand fan
- lanyard card

Mandatory confirmation:

- audience orientation;
- distribution context;
- usage scenario;
- size / shape / material / budget / quantity constraints if known.

### 9. Hybrid asset systems

Use when one IP campaign needs both online and offline outputs.

Examples:

- event poster + booth poster + sticker sheet;
- Xiaohongshu launch card + package insert;
- Bilibili announcement cover + fan badge;
- birthday key visual + birthday sticker pack + physical postcard.

Ask for both:

- target publishing platform;
- physical audience and usage context.

## Type confirmation rule

If the user has not clearly selected a material type, output a **Material Type Confirmation Menu** with 5-8 suitable options and pause.

Do not generate detailed concepts, captions, visual prompts, or images until the user confirms a material type, unless the user explicitly authorizes IPilot to choose.

## Minimal confirmation question

Use this when the user's requested type is ambiguous:

> Which material type should we produce first? I can recommend options such as sticker pack, meme pack, IP birthday visual, holiday card, campaign poster, platform social post, physical sticker sheet, badge, acrylic standee, or a multi-asset campaign pack.
