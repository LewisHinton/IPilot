# Guided Intake Questions

IPilot should use guided questions to complete the user's IP profile. Do not ask the full questionnaire at once. Ask only the next useful set of questions.

## Round 1 - Minimum Viable IP Brief

Use when the user has not provided enough information to begin.

1. What is the IP name?
2. What type of character is it? Animal, robot, object, food mascot, human-like figure, abstract symbol, or something else?
3. What fixed visual features must never change?
4. What personality should the IP express?
5. Do you already know which material type you want, or should IPilot recommend suitable material types?
6. Possible material types include sticker pack, meme pack, campaign poster, platform social post, IP birthday visual, holiday greeting card, physical sticker sheet, badge, acrylic stand, or a multi-asset pack. Which direction should we start with?
7. Is the output digital, physical, occasion-based, or hybrid?
8. If digital, which platform is the output for?
9. If physical, who is the intended audience and usage context?
10. If occasion-based, what occasion and publishing window is it for?

## Round 2 - Visual Consistency

Use when the IP visual identity is not specific enough.

1. What are the main and secondary colors?
2. What accessories or symbols must always appear?
3. What body shape, proportions, and face features are fixed?
4. What visual style should be used? Chibi, flat vector, 3D toy, hand-drawn, pixel art, clean illustration, or another style?
5. What visual changes are forbidden?
6. Do you have reference images? If yes, upload them before image generation.

## Round 2A - Reference Image Intake

Use when the user has uploaded, will upload, or mentions existing IP images.

1. Which image is the current official source of truth?
2. Are any images outdated, rejected, or only used as inspiration?
3. Which references control character identity, drawing style, pose, expression, or material layout?
4. Are there conflicts across references? Examples: different scarf color, different proportions, different line style, different accessory state.
5. What fidelity level is needed: strict, balanced, or loose?
6. Which reference-derived details must never drift? Examples: silhouette, color ratio, line style, visual anchors, face construction, accessory placement.
7. Which details may change? Examples: pose, expression, background, props, seasonal decorations, trend elements.

If multiple references conflict on identity-critical details, ask the user to choose before concept generation or image generation.

## Round 3 - Personality and Voice

Use when the character's behavior and copywriting style are vague.

1. Is the IP warm, energetic, professional, sarcastic, cute, rebellious, calm, or another type?
2. Is humor allowed? If yes, what kind?
3. Can the IP tease users, or should it only encourage them?
4. What words, tones, or expressions should it avoid?
5. Does it have catchphrases or repeated verbal habits?

## Round 4 - Brand and Audience

Use when brand context is weak.

1. What brand, project, or community does this IP represent?
2. Who is the target audience?
3. What user emotion should the asset trigger?
4. What user action should the asset encourage?
5. What topics must be avoided for brand safety?

## Round 5 - Digital Campaign and Platform Context

Use when the user wants promotional assets such as posters, announcement graphics, launch cards, social covers, video covers, carousel images, or event visuals.

1. What is the campaign topic?
2. What is the key message?
3. What should users do after seeing the asset?
4. What event time, deadline, discount, release date, or activity information must appear?
5. Which platform will this be published on: Xiaohongshu, WeChat, Bilibili, Douyin, X, Instagram, or another platform?
6. Which placement is it for: cover, carousel, story, video cover, article cover, dynamic image, group poster, launch card, profile banner, or another format?
7. Should this be a single-platform asset or a multi-platform adaptation pack?

If the platform is missing, stop and ask for it before detailed concept generation. Platform adaptation is part of the design, not a final afterthought.


## Round 5A - Asset Opportunity Discovery

Use when the user has an IP profile but does not know which material types to create.

1. What is the main goal: recognition, engagement, conversion, education, retention, or community activation?
2. Which platform matters most?
3. Should the output be more practical, more viral, more premium, more cute, or more humorous?
4. Are there material types you already tried and do not want repeated?
5. Should IPilot recommend a production path and ask you to choose?

IPilot should then output an Asset Opportunity Map or Material Type Confirmation Menu and pause for confirmation before detailed concept generation. Include occasion-based options when they fit the IP, such as birthday key visuals, holiday greeting cards, festival stickers, anniversary posters, seasonal cards, and countdown series.


## Round 5B - Physical Material Context

Use when the user wants tangible materials such as stickers, badges, acrylic stands, keychains, tote bags, T-shirts, postcards, packages, flyers, booth posters, coupons, cards, or standees.

1. What physical material do you want to make?
2. Who is the audience, and what style should it lean toward? Examples: children, students, fans, customers, employees, premium buyers, event visitors, open-source contributors.
3. How will it be distributed? Examples: paid product, free giveaway, event booth, purchase bonus, onboarding, fan merch, package insert, internal culture.
4. What is the usage scenario? Examples: laptop decoration, wearing, collecting, display, QR-code scanning, event check-in, product packaging.
5. Are there production constraints? Examples: size, shape, material, color limit, one-sided/two-sided, budget, quantity, deadline.
6. Are there physical safety requirements? Examples: child-safe, no sharp edges, no tiny detachable parts, washable material, durable outdoor use.

If audience orientation is missing, stop and ask for it before detailed physical concept generation. Physical material design depends on who receives and uses the object.

## Round 5C - Occasion-Based Asset Context

Use when the user wants IP birthday visuals, holiday greetings, festival posters, anniversary visuals, seasonal cards, countdown series, milestone celebration assets, or any date-specific IP material.

1. What is the occasion type? Examples: IP birthday, holiday, festival, anniversary, milestone, seasonal moment, school opening, exam season, graduation, year-end thank-you.
2. What is the date, date range, or publishing window?
3. What material type should this become: key visual, poster, greeting card, sticker pack, countdown series, physical postcard, or multi-asset campaign pack?
4. Which platform or channel will it be published on? Xiaohongshu, WeChat, Bilibili, Douyin, X, Instagram, offline, or multi-platform?
5. What tone should it have: warm, cute, celebratory, premium, funny, ceremonial, playful, nostalgic, or energetic?
6. Are there cultural, regional, religious, or brand-sensitivity constraints?
7. What mandatory text, date, hashtag, CTA, or slogan must appear?
8. Should the image contain text directly, or should text be handled as a separate design layer?

If the exact occasion output type is unclear, output a Material Type Confirmation Menu and pause before detailed generation.


## Round 5D - Deep IP Visual Consistency

Use when the user cares about stable IP appearance, provides a reference image, or reports that outputs drift.

1. What are the IP's strongest visual anchors? Examples: scarf, hat, emblem, eye shape, silhouette, forehead mark, costume, tail shape.
2. What line language should remain stable? Examples: thick rounded lines, thin clean vector lines, sketch lines, closed outlines, no outlines.
3. What drawing style should remain stable? Examples: flat vector, chibi, 3D toy, hand-drawn, pixel art, soft illustration.
4. What are the main color proportions? Example: orange 60%, cream 25%, blue 10%, yellow accent 5%.
5. What shape construction must remain stable? Examples: head-to-body ratio, eye size, limb length, accessory scale, silhouette.
6. What can freely change? Examples: pose, expression, scene, props, seasonal decorations, trend elements.
7. What visual quality failures must be rejected? Examples: broken lines, unclosed shapes, color bleeding, missing accessories, incomplete body, random text.

If the user does not know the exact design vocabulary, infer a first draft from the IP description or reference image and ask for confirmation.

## Round 5E - Text Layout and Typography

Use when the material includes posters, birthday visuals, holiday visuals, campaign cards, covers, or any text-heavy asset.

1. Which text is mandatory? Examples: title, subtitle, date, slogan, CTA, hashtag.
2. Should the text be rendered inside the generated image, or added later as a separate layout layer?
3. What font direction should it use? Examples: cute rounded, clean geometric, premium serif, tech monospaced accent, bold social display.
4. Where should text be placed? Examples: top title, bottom CTA, side label, blank sign held by IP, negative-space area.
5. Should IPilot recommend layout tools such as Figma, Canva, Photoshop, Illustrator, or Procreate?

Default to separate text layers when Chinese text accuracy matters.

## Round 5F - Trend Adaptation

Use when the user wants trend-based or topical assets.

1. What trend, meme format, platform habit, or topical moment should be referenced?
2. Which platform is this for?
3. Should the adaptation be obvious, subtle, parody-like, or only inspired by the trend?
4. Which parts of the trend should be adapted: emotion, composition, caption rhythm, scene, costume, or interaction style?
5. What should not be copied directly?
6. Should the IP remain visually strict, or can it wear temporary trend-themed props?

Support trend adaptation, but preserve IP identity and avoid unsafe or infringing copying.

## Round 6 - Image Generation Readiness

Use before invoking image generation.

1. Should the output be a sticker, square social post, poster, banner, transparent PNG-style asset, or another format?
2. Should text appear inside the image, or should it be handled as a separate caption layer?
3. How strictly should the image follow the reference image: strict, balanced, or loose?
4. How many image variants are needed?
5. Are there any elements that must not appear?
6. If references are used, are the source of truth and reference roles already confirmed?

## Questioning principle

Prefer small, useful rounds. If the user provides enough information, proceed. If one missing detail blocks generation, ask only that detail.

Use 1-3 questions per turn by default. Ask more only when the user requests a full brief or campaign plan.

When a question has predictable answers, present options instead of an open-ended prompt. Recommend a default when the user seems unsure.

When the Skill can infer a low-risk answer from the brief or reference image, state the inference and continue with a micro-confirmation. Stop and ask when the inference affects recognizability, rights, safety, platform fit, production cost, or image generation.

Before every major output, briefly recap what is known and what is assumed. This keeps the user in the loop without turning intake into a form.
