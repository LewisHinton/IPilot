# Occasion-Based IP Assets

Occasion-based assets are visuals tied to a birthday, holiday, festival, anniversary, seasonal moment, brand milestone, or recurring community ritual. They are common in IP operation because they give the character a living timeline and create predictable publishing opportunities.

## When to use

Use this module when the user asks for:

- IP birthday poster or birthday key visual;
- birthday sticker pack;
- holiday greeting card;
- festival visual;
- anniversary poster;
- seasonal campaign card;
- milestone celebration card;
- exam season / school opening / graduation asset;
- year-end thank-you card;
- countdown card for a special date.

## Required intake fields

Ask for these fields before detailed concept generation if missing:

1. Occasion type: birthday, holiday, festival, anniversary, milestone, seasonal moment, or another event.
2. Date or timing: exact date, date range, or publishing window.
3. Target platform: Xiaohongshu, WeChat, Bilibili, Douyin, X, Instagram, offline, or multi-platform.
4. Output type: key visual, poster, sticker pack, card, countdown series, physical postcard, or multi-asset pack.
5. Tone: warm, cute, celebratory, premium, funny, ceremonial, playful, nostalgic, or energetic.
6. Audience: fans, customers, students, community members, employees, contributors, children, or general public.
7. Cultural constraints: festival customs, taboo imagery, religious or regional sensitivity, language requirements.
8. Mandatory text: slogan, date, hashtag, CTA, event name, QR-code placeholder, or birthday message.

## Recommended output patterns

### IP birthday key visual

Use for annual celebration or character operation.

Output should include:

- main birthday concept;
- IP pose and expression;
- birthday symbol system: cake, candle, ribbon, gift, party hat, confetti, stage, fan letters;
- visual hierarchy;
- caption / slogan;
- platform adaptation notes;
- image prompt;
- safety and consistency notes.

### Birthday countdown series

Use when the user wants a campaign rather than one image.

Typical sequence:

1. D-7 teaser card
2. D-3 memory / story card
3. D-1 countdown card
4. Birthday key visual
5. post-event thank-you card

### Holiday greeting card

Use for public greetings, seasonal brand warmth, or community interaction.

Output should include:

- holiday symbol set;
- cultural fit notes;
- IP action;
- greeting copy;
- platform-specific visual constraints;
- risk notes.

### Festival sticker pack

Use for chat and community engagement.

Output should include 6-12 stickers with:

- emotion;
- gesture;
- short caption;
- festival element;
- prompt;
- safety note.

## Cultural and brand safety notes

Occasion-based assets can create unexpected risk. Check:

- whether the holiday has religious, political, regional, or historical sensitivity;
- whether symbols are being used respectfully;
- whether the IP's humor is too casual for the occasion;
- whether the event is suitable for the target audience;
- whether dates and greeting language are accurate;
- whether commercial CTAs are inappropriate for solemn occasions.

## Confirmation rule

After generating 2-4 occasion creative directions, ask the user to confirm:

- which direction to use;
- whether to make a single asset or a series;
- whether to proceed to image generation or only output prompt/copy packs.
