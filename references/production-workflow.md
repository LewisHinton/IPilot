# Production Workflow

IPilot is designed as an end-to-end IP asset production workflow.

It covers:

1. IP intake;
2. material type routing;
3. human-in-the-loop dialogue and micro-confirmations;
4. platform / audience / occasion questions;
5. deep IP consistency profiling;
6. creative direction;
7. user confirmation;
8. asset concept generation;
9. brand safety review;
10. image-generation readiness review;
11. image generation or prompt handoff;
12. post-generation quality review;
13. revision prompts;
14. text layout guidance;
15. production handoff notes.

The workflow should advance in short loops: clarify the next decision, produce the next useful artifact, ask for confirmation when the next step changes the output, and continue only after the user confirms or explicitly authorizes auto-pick.

For physical materials, IPilot provides practical design and handoff guidance. It does not replace the final supplier-specific production checklist. Users should confirm final DPI, bleed, cut lines, material, color mode, file format, and manufacturing constraints with their vendor.

## Final package structure

A complete asset production package should include:

```markdown
# Final Asset Production Package

## 1. Project Summary
## 2. Material Route and Confirmed Asset Type
## 3. Deep IP Consistency Profile
## 4. Platform / Audience / Occasion Constraints
## 5. Creative Direction
## 6. Approved Asset Concepts
## 7. Image Generation Prompts
## 8. Negative Prompts / Constraints
## 9. Text Layout Plan
## 10. Font Direction
## 11. Safety and Consistency Review
## 12. Generated Image Review
## 13. Revision Instructions
## 14. Production Handoff Notes
```

## Handoff notes for physical materials

For physical materials, include:

- intended material type;
- audience orientation;
- usage scenario;
- approximate size preference;
- shape / cut expectation;
- one-sided or two-sided design;
- text placement;
- color limitation, if any;
- durability expectation;
- vendor confirmation reminder.
